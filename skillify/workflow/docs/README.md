# Skillify 文档基线

本目录是 Skillify 项目的正式基线文档组，用来替代会话记忆，避免产品、设计、实现逐步漂移。

## 文档职责

- `01-product-requirements.md`
  - 定义产品定位、目标用户、核心链路、业务规则、功能范围、页面级验收标准。
- `02-ui-design-spec.md`
  - 定义逐页界面说明、视觉方向、布局结构、交互规则、状态设计、响应式原则。
- `03-technical-architecture.md`
  - 定义当前原型的实现基线、未来真实服务演进边界、技术约束和替换策略。

## 优先级

当三份文档出现冲突时，按以下顺序判断：

1. 产品文档决定做什么、不做什么。
2. UI 文档决定页面如何表达和交互。
3. 技术文档决定如何实现，以及实现边界。

## 使用规则

- 日常小修小补可以按当前上下文推进，但不得违背基线文档。
- 重要里程碑开始前，主代理必须重新读取三份基线文档。
- 重要里程碑固定为：
  - 新的大功能或新页面簇启动前
  - 涉及架构边界变化的实现前
  - 接入真实外部服务前
  - QA、验收、准备发布前
- 里程碑汇报必须说明：
  - 本轮引用了哪些基线文档
  - 是否发现产品、设计、技术之间不一致
  - 若不一致，应更新哪份文档

## 更新顺序

文档需要调整时，默认按下面顺序处理：

1. 先确认产品文档是否变化。
2. 再更新 UI 文档中的页面和交互说明。
3. 最后更新技术文档中的实现边界和替换策略。
4. 若有流程门禁变化，同步更新项目根目录下的 `README.md`、`AGENTS.md`、`WORKFLOW.md`。

## 当前定位

- 文档默认全中文。
- 技术文档采用“双轨文档”写法。
- 当前站点仍以本地可演示原型为主。
- 源码中的中文乱码不作为产品语义基线，以本目录文档为准。
