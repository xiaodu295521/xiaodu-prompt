# Skillify 示例工作流

这份文件展示一个真实项目如何把公共模板落到执行顺序里。

## 默认顺序

1. 识别 `Skillify-old` / `Skillify-new`
2. 先完成 `new -> old` 基线快照
3. 回看基线文档
4. 如属新阶段，先走 PRD -> UI -> 技术文档
5. 再进入任务拆解、派单和实现
6. 每轮结束做验证和汇报

## 里程碑回看

以下节点开始前，重新读取基线文档：

- 新的大功能启动前
- 新页面簇启动前
- 架构边界变化前
- 接入真实外部服务前
- QA、验收、准备发布前

## 典型输出

每轮较大工作应至少说明：

1. 本轮目标
2. 是否已完成 `new -> old` 快照
3. 引用了哪些基线文档
4. 派了哪些子代理
5. 改了什么
6. 如何验证
7. 还剩哪些风险
