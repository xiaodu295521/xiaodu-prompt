# 仓库结构说明

## 顶层目录

### `workflow/`

放全局可复用规则。

适合放：

- 全局 `AGENTS`
- 全局 `WORKFLOW`
- 全局技能定义
- 通用工作流资产模板

### `templates/`

放用户可以直接复制的模板。

适合放：

- 全局模板
- 项目模板
- 文档模板

### `docs/`

放公开说明文档。

适合放：

- 快速开始
- 仓库结构说明
- 自定义指南
- 升级指南

### `examples/`

放真实案例。

适合放：

- 一个完整示例项目
- 示例项目的规则文件
- 示例项目的文档基线

## 为什么要把模板和示例分开

模板的目标是“让别人直接复制使用”。

示例的目标是“让别人看懂模板在真实项目里如何落地”。

如果两者混在一起，第一次打开仓库的人会分不清：

- 哪些是必须复制的
- 哪些只是案例
- 哪些是项目专属内容

## 当前推荐阅读顺序

1. 根 `README.md`
2. `docs/getting-started.md`
3. `templates/`
4. `workflow/`
5. `examples/skillify/`

## 不应该出现在公开主线里的内容

这些内容不应该放在公共模板仓库主入口：

- 个人本地绝对路径
- 个人专用同步说明
- 个人工作缓冲目录语义
- 只适合单个项目的内部说明

这类内容如果确实需要保留，建议放在你自己的私有附录里，而不是公开仓库。
