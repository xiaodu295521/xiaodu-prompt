# 自定义指南

这份文档讲的是：拿到模板后，哪些地方必须替换，哪些地方可以保留。

## 必须替换的占位符

### 项目名相关

- `<YOUR_PROJECT>`
- `<YOUR_PROJECT_NEW>`
- `<YOUR_PROJECT_OLD>`

用途：

- 标记你的项目名称
- 标记双目录名称

### 环境相关

- `<YOUR_CODEX_HOME>`

用途：

- 标记你本机的 Codex 配置目录

### 备份相关

- `<YOUR_BACKUP_REPO_URL>`

用途：

- 标记你自己的远端备份仓库地址

## 哪些内容通常可以保留

- “先 PRD、再设计、再技术文档”的顺序
- “先把 `new` 同步到 `old`，再继续开发”的规则
- “主代理编排、子代理执行”的协作方式
- “对用户尽量说人话”的沟通要求

这些一般是方法论，不强依赖具体项目。

## 哪些内容通常要按项目改

- 用户是谁
- 核心业务链路是什么
- 页面有哪些
- 技术边界在哪里
- 需要哪些角色和技能

## 自定义建议顺序

1. 先改项目名称和目录名称
2. 再改全局路径
3. 再改远端仓库地址
4. 再改项目示例中的具体业务描述

## 不建议直接复制的内容

如果你不是在做和 Skillify 非常接近的项目，不建议直接复制：

- Skillify 的页面定义
- Skillify 的产品定位
- Skillify 的案例文档正文

这些更适合参考结构，而不是原文照抄。
