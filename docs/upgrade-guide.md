# 升级指南

这份文档说明：当模板仓库更新后，如何把新规则平稳同步到你自己的环境。

## 升级原则

- 先看变更说明
- 再决定是否合并
- 不要直接把新版本覆盖到正在开发的项目里

## 推荐升级顺序

1. 拉取模板仓库最新内容
2. 先看 `README` 和 `workflow/WORKFLOW.md`
3. 再看全局 `AGENTS` 和技能文件是否有规则变化
4. 再看 `templates/` 里有没有模板结构变化
5. 最后决定是否把变化同步到你自己的项目

## 升级项目时的建议做法

对于已经在跑的项目：

1. 先把当前项目完整保存到 `-old`
2. 在 `-new` 里单独合并规则变化
3. 检查是否会影响：

- 文档顺序
- 子代理并发
- 用户沟通方式
- 备份流程

## 哪些变化要谨慎升级

- 目录结构变化
- 规则优先级变化
- 子代理并发策略变化
- 文档门禁变化
- 备份流程变化

## 判断要不要升级的简单标准

适合升级：

- 当前项目还在早期
- 规则变化会明显减少混乱
- 你希望以后项目都统一成新方式

可以暂缓：

- 当前项目已经进入后期
- 规则变化会打断现有节奏
- 新规则只是文案整理，不影响主流程
