# 更新说明

## 2026-04-08

### 重磅更新！

- 全局工作流升级为“先识别项目类型，再套稳定启动方案”。
- Web 项目默认提醒采用 `方案 B：稳定内核 + 可换技术栈`。
- 非 Web 项目默认分流到移动应用、桌面工具、自动化或 bridge 服务三类稳定方案。
- 文档阶段新增“未来高概率扩展对齐”要求，PRD 和技术文档都要提前预留下一阶段边界。
- 目录策略升级为“中等预留”，避免把所有功能都堆在根目录。
- 主代理编排规则继续沿用低并发、分阶段派单和 `429` 降级机制。
- Web 项目默认交付方式正式固定为：
  - 最小 CI
  - 预览环境
  - 正式环境手动发布
- Skillify 示例、全局规则和项目模板已经同步到仓库，后续可直接按模板复用。

### 公共入口文档清理

- 重写了仓库根 `README.md`，明确公共模板库定位。
- 重写了 `docs/getting-started.md`、`docs/repo-structure.md`、`docs/customization-guide.md`、`docs/upgrade-guide.md`。
- 重写了 `examples/skillify/README.md`，明确 Skillify 是示例，不是模板本体。
