## Global AGENTS Template

把这份文件复制到你自己的全局规则位置后，再替换占位符。

### 基础占位符

- `<YOUR_CODEX_HOME>`
- `<YOUR_BACKUP_REPO_URL>`

### 全局默认要求

- 新项目默认采用双目录：`<YOUR_PROJECT>-old` / `<YOUR_PROJECT>-new`
- 每轮进入开发前，先把 `-new` 的完整基线同步到 `-old`
- 所有实际开发只在 `-new`
- 新项目或新阶段先走：PRD -> 设计文档 -> 技术文档
- 实施计划、汇报和风险说明默认使用通俗中文
- 中大型任务默认采用“主代理编排，子代理执行”

### 项目类型分流

- Web 项目：
  - 主代理必须主动提醒采用 `方案 B：稳定内核 + 可换技术栈`
  - 重点说明：目录、模块边界、contracts、测试门禁和 review 规则固定
  - 默认交付策略为：`最小 CI + 预览环境 + 手动正式发布`
- 非 Web 项目：
  - 默认先判断属于移动、桌面还是自动化 / bridge
  - 再给出对应稳定方案

### 文档阶段要求

- PRD 阶段必须和用户对齐：
  - 当前范围
  - 下一阶段范围
  - 暂不实现但要预留的能力
- 技术文档阶段必须写清：
  - 未来模块演进图
  - 目录预留方案
  - CI/CD 策略

### 目录预留规则

- 默认采用“中等预留”
- 只预留高概率模块
- 不预留纯猜测模块
- 每个预留目录都必须有明确用途
- 根目录层级要克制，不允许所有功能堆在根目录

### 子代理默认规则

- 默认并发 `2`
- 仅在安全条件满足时短时升到 `3`
- 首次 `429`：等待后重试一次
- 第二次 `429`：降为串行

### 规则备份

- 工作流文件更新后，先同步到本地规则仓库或本地缓冲区
- 默认先做本地保存
- `<YOUR_BACKUP_REPO_URL>` 只在用户明确确认后再 push
