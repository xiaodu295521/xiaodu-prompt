# Global WORKFLOW Template

## 目标

这份工作流用来统一说明：

- 项目如何启动
- 项目类型如何分流
- 文档如何先行
- 目录如何预留
- 子代理如何分工

## 默认流程

1. 识别 `<YOUR_PROJECT>-old` / `<YOUR_PROJECT>-new`
2. 判断项目属于 Web、移动、桌面，还是自动化 / bridge
3. 如果是 Web，主动提醒采用 `方案 B：稳定内核 + 可换技术栈`
4. 如果是 Web，默认采用 `最小 CI + 预览环境 + 手动正式发布`
5. 执行 `new -> old` 基线快照
6. 新阶段先做 PRD、设计文档、技术文档
7. 在 PRD 和技术文档里对齐未来高概率扩展需求
8. 技术文档确认目录预留方案
9. 再进入拆解、派单和实现
10. 每轮结束给出验证结果

## 文档要求

- PRD 固定写：
  - 当前范围
  - 下一阶段范围
  - 暂不实现但要预留的能力
- 技术文档固定写：
  - 未来模块演进图
  - 目录预留方案
  - CI/CD 策略

## 目录预留要求

- 默认采用中等预留
- 只预留高概率目录
- 不把大量空目录一次性铺满
- 根目录保持克制

## 用户沟通

- 默认使用通俗中文
- 先讲结果，再讲原因，最后讲实现
