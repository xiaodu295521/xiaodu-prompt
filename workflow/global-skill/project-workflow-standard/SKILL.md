# Project Workflow Standard

## 默认行为

- 每个项目先准备 `<YOUR_PROJECT>-old` 和 `<YOUR_PROJECT>-new`
- 所有改动只在 `-new`
- `-old` 保存当前轮次开始前的完整版本

## 基础规则

1. 开始实施前先识别双目录
2. 信息不足时，先说缺什么、为什么缺、建议默认方案是什么
3. 新项目或大阶段启动前，先做 PRD、设计文档、技术文档
4. 用户看得见的说明默认用通俗中文
5. 每轮都要有验证结论

## `new -> old` 快照规则

每次进入以下阶段前，先把 `-new` 的完整内容复制到 `-old`：

- 新一轮开发
- 较大修复
- 发布前收口

## 备份建议

如果你需要把规则文件同步到云端仓库，请把下列占位符替换成自己的实际值：

- `<YOUR_BACKUP_REPO_URL>`
- `<YOUR_CODEX_HOME>`

## 每轮输出建议

1. 本轮目标
2. 是否已完成 `new -> old` 快照
3. 信息是否足够
4. 本轮改了什么
5. 验证结果
6. 风险和待确认项
